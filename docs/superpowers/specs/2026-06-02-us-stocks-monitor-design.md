## 一、项目概述

个人使用的美股自选股监控系统。每日早8:00自动从akshare拉取美股日线数据，计算MA/KDJ指标，生成静态JSON，前端纯静态页面展示。通过pushplus推送每日晨报到微信。

## 二、技术选型

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端 | 纯静态 HTML/CSS/JS | 无框架，移动端响应式，托管GitHub Pages |
| 数据拉取 | Python (akshare) | 美股日线 + 东方财富行业/市值 |
| 定时任务 | GitHub Actions | UTC 00:00 (= 北京 08:00) triggered workflow |
| 数据存储 | stocks.json (静态文件) | ~80KB for 200 stocks |
| 推送服务 | pushplus | HTTP POST，无需认证 |
| 自选股管理 | localStorage | 浏览器本地存储 |


## 五、关键交互决策

| 交互点 | 方案 |
|--------|------|
| 主页布局 | C：信号区 + 行业手风琴 |
| 股票详情 | A：横向分区卡片（价格行 → 四格指标 → 均线条 → KDJ条） |
| 自选股管理 | A：顶部搜索框 + 列表，支持代码即搜即加，右侧删除 |
| 顶部导航 | B：双行（标题+按钮行 / 时效提示行） |
| 手动刷新 | C：默认重新fetch JSON（免费），按需调GitHub API强制更新（限10次/日） |

## 六、数据流

```
[用户访问] → fetch stocks.json → 解析分组/排序/信号 → 渲染页面

[手动刷新] → fetch stocks.json (检查是否有新数据)
          → 如需强制更新 → 调GitHub API触发Actions → 轮询等待 → 更新页面

[每日08:00] → Actions启动 → Python: akshare拉取 → 计算MA5/MA20/KDJ
            → 写入stocks.json → git push → Pages自动部署 → pushplus推送
```
